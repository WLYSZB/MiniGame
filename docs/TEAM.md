## 协作规范

### Git 分支策略

```
main          - 稳定版本，可运行
├── develop   - 开发主线
│   ├── feature/xxx  - 新功能
│   └── bugfix/xxx   - 修复
```

### 提交规范

```
feat: 新功能
fix: 修复bug
art: 美术资源更新
docs: 文档更新
```

### 工作流程

1. 从 `develop` 创建功能分支
2. 完成后提交 Pull Request
3. 至少1人 Review 后合并
4. 定期将 `develop` 合并到 `main`
